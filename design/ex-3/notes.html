Exercise 3: Basics of Icons

Challenges Faced:

Icons seemed easy at first, but styling them and making everything look consistent was surprisingly difficult. My first attempt at adding icons using Font Awesome worked fine for Level 1, but by the time I reached Level 3, things started falling apart. The alignment was messy, spacing felt uneven, and my navigation bar didn’t look professional.

Working with SVGs was also tricky. I struggled to combine inline SVGs with icons from the library while maintaining a clean layout. Hover effects were functional, but the animations felt awkward.

Tools and Resources Used:

  • Font Awesome for icon fonts
  • Inline SVG files for custom icons
  • CSS for icon styling, hover effects, and layout

My Thought Process:

I started with the basics—adding three icons using Font Awesome. It worked smoothly, and I could size and color them using CSS. For Level 2, I explored inline SVGs, which gave me more flexibility but also introduced layout issues. Mixing font-based icons and SVGs in the grid required extra effort to ensure they looked uniform.

Level 3 was the real challenge. Designing a navigation bar with icons as the main elements sounded simple, but I underestimated the alignment struggles. I tried to position icons and text labels using CSS, but the spacing wasn’t consistent, and the overall look felt unpolished. For buttons with embedded SVGs, I managed to get the icons to display, but aligning them within the buttons didn’t come out as clean as I wanted.

Specific Challenges:

  • Combining inline SVG icons with Font Awesome icons in a uniform layout.
  • Creating hover effects that were smooth and visually appealing.
  • Aligning icons and labels in the navigation bar properly.
  • Ensuring consistent spacing between grid items and buttons.

Key Learning Outcomes:

  • How to use icon libraries like Font Awesome and style icons with CSS.
  • Basic implementation of inline SVGs for custom icons.
  • Understanding the importance of consistent spacing and alignment for better design.
  • Experimenting with hover effects to add interactivity.

What Could Be Improved:

  • Better alignment and spacing of icons, especially in the navigation bar.
  • Smoother and more visually appealing hover effects.
  • Improved consistency between font-based and SVG icons.
  • Using a more structured approach to grid layouts for uniformity.

Reflection:

This exercise was a reality check for me. I realized that even something as small as icons can make or break the overall design. While I managed to complete all levels, I wasn’t happy with how the navigation bar and buttons turned out. I definitely need to focus more on alignment and spacing. Moving forward, I’ll explore frameworks like Flexbox or Grid to handle layouts better and pay closer attention to design details.

On a positive note, working with SVGs gave me a better understanding of their flexibility, and Font Awesome was a great tool to learn icon styling. Overall, I’ve identified areas for improvement and feel motivated to keep practicing!

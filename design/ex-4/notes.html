Exercise 4: Basics of Images

Challenges Faced:

Working with images was moderately tricky. I thought it would be simple, but maintaining image proportions and ensuring they looked good across devices required extra effort.

For Level 1, using placeholder services like Lorem Picsum was straightforward, but styling the images with consistent width, height, and border-radius took trial and error. I initially faced an issue where images looked stretched due to incorrect aspect ratios.

Level 2 brought new challenges. Adding a full-bleed header image was exciting, but applying overlays using gradients required careful tweaking to avoid affecting the text readability.

Level 3 tested my understanding of responsive design. Implementing the <picture> element to load different image versions for varying screen sizes was a bit confusing at first. Optimizing image loading with lazy loading was also something new for me, and I had to read about how it actually works.

Tools and Resources Used:

  • HTML5 for image structure
  • CSS for image styling (size, border-radius, overlays)
  • Lorem Picsum for placeholder images

My Thought Process:

I started by adding simple images and adjusting their width and height using CSS. The border-radius styling gave the images a modern look, which I liked.

For the full-bleed header image in Level 2, I experimented with CSS overlays using gradients. I wanted to keep the focus on the image while making sure the overlaid text was readable. Finding the right balance for opacity and contrast took time.

At Level 3, I had to focus on performance and responsiveness. The <picture> element helped me serve optimized images for smaller devices. I also learned how lazy loading works and implemented it to speed up page performance. While it worked, I still feel I need to practice this more.

Specific Challenges:

  • Maintaining image proportions without distortion.
  • Creating effective overlays without reducing readability.
  • Understanding and implementing the <picture> element.
  • Using lazy loading to optimize performance.

Key Learning Outcomes:

  • Basic image styling with CSS (width, height, and border-radius).
  • Adding full-bleed images with gradient overlays for headers.
  • Implementing responsive images using the <picture> element.
  • Lazy loading to improve page load performance.

What Could Be Improved:

  • More refined gradient overlays for better visual appeal.
  • Improving the understanding of responsive image techniques.
  • Exploring tools for automated image optimization.
  • Practicing lazy loading with real-world examples.

Reflection:

This exercise showed me that handling images in web design involves more than just adding them to the page. I learned a lot about responsiveness, optimization, and performance improvements.

I’m still not completely confident with the <picture> element and lazy loading, but I’m glad I got a chance to experiment with them. In the future, I’d like to explore automated tools for optimizing images to make the process more efficient.

Thought Process for Exercise 1

Level 1: Semantic HTML and Structure

During the first part of this exercise, I focused on using semantic HTML elements to structure the page. One of the questions I had was, "Why should I use semantic tags like <header>, <footer>, and <article> instead of <div> elements?" I learned that semantic tags help make the page more readable for both humans and search engines, which improves SEO and accessibility.

Another question that came up was, "How do I structure my page using <section> and <article> to separate different parts of content?" Using these elements, I was able to create a clean, understandable layout that made sense both for users and for any future development.

Level 2: Layout and Styling

When implementing the golden layout structure with two sidebars and a main content area, I asked myself, "How can I ensure that my layout works well across different screen sizes?" This led me to use a combination of CSS Grid and Flexbox. I used Grid to create a responsive layout for larger screens and Flexbox for smaller devices, ensuring the layout adapts without breaking.

Additionally, I wondered, "How do I create a simple navigation menu inside my sidebars?" I used <nav> and styled the links inside it to make the menu clear and easy to navigate.

Level 3: Adding Content and Optimizing

As I added content, I started considering, "How do I ensure that my images are accessible?" I added alt text to the image elements to improve accessibility, making sure that users with visual impairments could understand the content. I also wondered, "How can I ensure that my table looks clean and readable?" I used <th> for header cells and styled the table to make it more user-friendly.

Finally, I thought about, "How can I improve the readability of my text and content?" I made sure to structure paragraphs and content logically so that users could easily scan through the page. I also used clear headings to guide the user through different sections of the page.

Reflection

This exercise helped me understand the importance of structuring HTML elements semantically for both accessibility and SEO. I learned how using proper tags like <section>, <article>, and <nav> can make the page more understandable and user-friendly. I also learned how to create a balanced and responsive layout using Grid and Flexbox, ensuring the page adapts to different screen sizes.

Overall, the exercise emphasized how small details like accessible images, well-structured tables, and logical content organization can make a big difference in the quality of a web page.

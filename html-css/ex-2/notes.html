Thought Process for the Exercise 2

Level 1: Define CSS Variables and Apply Basic Styling

To begin, I thought about how CSS variables could make the design more consistent and reusable. I defined variables for primary and secondary colors, font sizes for headers and body, and spacing for margins. These were included in a separate variables.css file linked to the main CSS.

To understand the implementation better, I asked AI: "How do I define CSS variables and use them in a project?" The response helped me use the :root selector for defining variables and var() to reference them, making the setup seamless.

Level 2: Using Default Values and Styling Alternate Rows

Next, I worked on ensuring fallback styles with default values for CSS variables, such as var(--primary-color, black). This helped avoid unexpected issues when variables were undefined.

For table styling, I used the nth-child selector to create alternating row colors. AI clarified this approach when I asked: "How do I style every alternate row in a table?" It worked perfectly to improve table readability.

Level 3: Dynamic Theme Switching with Variables

Dynamic theming was an exciting challenge. I wanted to toggle between light and dark themes using CSS variables. AI suggested toggling class names on the <html> or <body> tags to redefine variables for each theme.

By following this, I implemented a simple JavaScript function to handle the theme switch. This made the page visually adaptable to user preferences with minimal CSS adjustments.

Reflection

This exercise deepened my understanding of CSS variables and their potential for creating consistent, flexible, and theme-able designs. AI was instrumental in breaking down concepts like default values, advanced selectors, and theme switching into manageable steps, enabling me to approach each level effectively.
